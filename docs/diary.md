# Дневник наблюдений проекта — TellMeApp

---

## [2026-05-07] — Проектирование архитектуры

### Наблюдения

- Проект представляет собой фоновое приложение для голосового ввода текста в любые поля ввода на устройстве
- AquaVoice API (Avalon) полностью совместим с OpenAI Whisper API — это упрощает интеграцию, можно использовать стандартный подход с Retrofit multipart
- Приложение Happ использует минималистичный дизайн с центральной кнопкой (power button metaphor) и тёмной цветовой схемой — это хороший ориентир для UI
- Основные технические сложности: (1) глобальный перехват нажатий кнопки громкости, (2) вставка текста в чужие приложения, (3) стабильная фоновая работа

### Решения

- **Архитектура**: MVVM + Clean Architecture — разделение ответственности между слоями, удобное тестирование
- **Перехват кнопки**: AccessibilityService с `onKeyEvent` — даёт возможность и перехватывать кнопки, и вставлять текст через один сервис
- **Фоновая работа**: ForegroundService с persistent notification — единственный надёжный способ для Android 8+
- **API-клиент**: Retrofit + OkHttp для multipart-запросов к Whisper-совместимому API
- **Хранение**: DataStore Preferences для настроек и данных подписки (легковесное решение)
- **DI**: Hilt — стандартный выбор для Android, интеграция с ViewModel и Service

### Проблемы

- Точные детали AquaVoice API находятся за авторизацией в дашборде — пришлось опираться на публичные данные и совместимость с OpenAI Whisper
- API подписок ещё не готов — в архитектуре предусмотрены mock-реализации для UI-демонстрации
- На некоторых Android-версиях (особенно китайские прошивки) AccessibilityService может быть агрессивно убит системой — нужен механизм восстановления
- Перехват кнопок громкости через AccessibilityService может конфликтовать с другими приложениями, использующими этот механизм
