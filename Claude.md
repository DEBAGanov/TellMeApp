# Claude — Правила проекта TellMeApp

## Общие правила

1. **Перед каждой операцией** — читать этот файл и актуальную документацию в `/docs/`
2. **Перед изменением кода** — читать `docs/project.md` для понимания архитектуры
3. **После каждого шага** — обновлять `docs/tasktracker.md` (статус задач)
4. **При значительных изменениях** — обновлять `docs/project.md` и `docs/changelog.md`
5. **При принятии технических решений** — записывать в `docs/diary.md`
6. **При возникновении вопросов** — добавлять в `docs/qa.md`

## Архитектурные стандарты

- **Паттерн**: MVVM + Clean Architecture (domain/data/ui слои)
- **DI**: Hilt
- **UI**: Jetpack Compose + Material 3
- **Навигация**: Compose Navigation
- **Сеть**: OkHttp + Retrofit
- **Хранение**: DataStore Preferences
- **Сериализация**: Kotlinx Serialization

## Кодирование

- Язык: Kotlin
- Стиль: Kotlin Coding Conventions
- Минимум комментариев — код должен быть самодокументируемым
- Каждый файл начинается с заголовка:
```kotlin
/**
 * @file: [имя файла]
 * @description: [краткое описание]
 * @dependencies: [связанные компоненты]
 * @created: [дата]
 */
```
- Принципы: SOLID, KISS, DRY
- Не оставлять неиспользуемый код, закомментированные блоки, TODO

## Документация

- `docs/project.md` — архитектура, стек, структура, API
- `docs/tasktracker.md` — задачи с приоритетами и статусами
- `docs/changelog.md` — хронология изменений
- `docs/diary.md` — технический дневник
- `docs/qa.md` — открытые вопросы

## Процесс

1. Перед шагом → подтверждение пользователя
2. После шага → резюме (до 5 пунктов) + обновление tasktracker
3. При проблемах → 2-3 альтернативных подхода
4. Code review для всех изменений
